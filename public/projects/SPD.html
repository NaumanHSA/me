üåü Project Overview

Real-Time Detection of a Suspicious Person in CCTV Footage is a cutting-edge computer vision system designed to automatically detect potentially dangerous individuals in live CCTV streams. The system focuses on identifying persons holding weapons‚Äîa critical sign of potential threats‚Äîthus reducing reliance on human operators and enabling proactive security measures in real-time.

The project addresses key challenges in traditional CCTV surveillance, including:

  • High false alarm rates in existing automated systems.
  • Difficulty detecting small objects like weapons in low-resolution footage.
  • Need for real-time processing to respond to threats as they happen.

This work was conducted as a research project at the Department of Computer Science, Air University, Islamabad.

üéØ Problem Definition

  • Suspicious Person Definition: For this project, a suspicious person is defined as any person detected holding a visible weapon.
  • Scenario: Indoor/outdoor CCTV footage at entrances, exits, corridors, or public spaces.

üß© Architecture and Workflow of the Suspicious Person Detector (SPD)

The Suspicious Person Detector (SPD) is designed as a carefully structured pipeline that balances accuracy, speed, and low false alarms in detecting threats from CCTV streams. The end-to-end workflow can be understood by analyzing the following diagrams, each representing critical stages of the system.

1. Overall System Architecture

The SPD system is built around three interconnected modules:

  • Front-End GUI (Angular):
    • Displays live video streams and overlays detection results in real time.
  • Server Layer:
    • Manages communication between the front-end and back-end using WebSockets for low-latency data transfer.
  • Back-End (Python):
    • Handles video frame acquisition, detection logic, and suspicious person classification.

Figure 1: Modular Parts of the SPD System

2. Detailed Design Methodology

The detection pipeline within the back-end involves several sequential stages:

  1. Frame Extraction:

    • Video stream is captured from the CCTV feed, frame by frame.

  2. Motion Detection (Optional Idle Mode):

    • A lightweight motion detection algorithm checks whether there is significant motion between consecutive frames.
    • Helps avoid running heavy computations on static scenes.

  3. Object Detection with YOLOv3:

    • Detects bounding boxes for:
      • Persons
      • Weapons
    • Runs on high-resolution frames for better detection of small objects like handguns.

  4. Suspicious Person Classification:

    • Matches detected weapon boxes with person boxes using spatial and statistical analysis.

Figure 2: Design Methodology of the SPD System

3. Classifier Logic for Suspicious Person Detection

Once the object detection model predicts bounding boxes, the system performs suspicious person classification using:

  • IoW (Intersection over Weapon):
    • Measures overlap between weapon and person bounding boxes.
  • Confidence Scores:
    • Utilizes YOLO‚Äôs confidence for both weapon and person detections.
  • WoP Ratio (Weapon over Person):
    • Helps disambiguate overlapping boxes when a weapon might appear associated with multiple persons.
  • AAIFc (Alarm Activation Interval Frame Count):
    • Ensures a suspicious person is detected across multiple consecutive frames (e.g. 3 frames) before raising an alarm.

This pipeline reduces false alarms while maintaining high detection accuracy.

üö¶ End-to-End Workflow Summary

Here‚Äôs how the entire system works together in real time:

  1. CCTV stream is captured and divided into frames.
  2. Each frame optionally passes through motion detection.
  3. YOLOv3 processes the frame to detect weapons and persons.
  4. The SPD algorithm:
    • Matches weapon and person bounding boxes.
    • Computes IoW and CSP scores.
    • Filters out false matches using WoP ratios.
    • Checks for consistent detections across multiple frames (AAIFc).
  5. If confidence exceeds thresholds:
    • The system labels the frame as containing a suspicious person.
    • Alerts are sent via the server to the GUI for real-time display.

This layered approach ensures:

  • Minimal false alarms.
  • Real-time performance, even on modest GPUs.
  • Reliable detection of threats in live CCTV feeds.

üîç Suspicious Person Detection Algorithm

Once bounding boxes for persons and weapons are predicted:

  1. Weapon-Person Pairing (IoW):

    • Calculates Intersection over Weapon (IoW) between bounding boxes.
    • Higher IoW indicates a higher probability the person is holding the weapon.

    $$ IoW = \frac{\text{Area of Overlap}}{\text{Area of Weapon Box}} $$

  2. Composite Confidence Score (CSP):

    • Combines:
      • IoW value
      • Confidence score of weapon detection
      • Confidence score of person detection
    • Averaged to determine the likelihood of a suspicious person.

    $$ CSP = \frac{IoW + C_{weapon} + C_{person}}{3} $$

  3. Multiple Overlap Resolution (WoP):

    • In cases where a weapon overlaps with multiple people:
      • Calculates Weapon-over-Person ratio (WoP) for each pair.
      • Filters out unlikely pairings by comparing against observed ratio thresholds.

  4. Alarm Activation Interval (AAIFc):

    • Alarm only triggers if the same suspicious person is detected in 3 consecutive frames to reduce false alarms.

üìà Dataset Preparation

  • Final dataset comprised 2,789 images:

    • 1,140 person images
  • Sources:
    • Internet downloads

‚ö° Speed Optimization

Problem: Running YOLO on every frame was computationally expensive and caused dropped frames in live feeds.

Solution: Motion Detection Pre-Check

  • Lightweight frame differencing algorithm detects motion before feeding frames into YOLO.
  • Reduces unnecessary computation on static scenes (e.g., at night).
  • Helps maintain real-time performance.

üéØ Experimental Results

  • Tested on 60 scenes:

    • 10 scenes without suspicious persons
  • Achieved:
    • Correct detection in 48 out of 50 positive scenes.
    • No false positives in negative scenes.
  • Resulting in an overall system accuracy of 96% for real-time suspicious person detection.
